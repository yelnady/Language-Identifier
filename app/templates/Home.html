Overview of the model

The Dataset

Our dataset consists of 9,699,409 sentences spanning over 398 different languages . 

Out of them, we chose 11 languages: English, Spanish, French, German, Italian, Portuguese, Arabic, Vietnamese, Mandarin, Dutch, and Romani. 




Preparing the Dataset

We created thresholds as limits for how long and how short each sentence can be. By the end of that, our data was down to 4,274,241 sentences.

But, as shown in the first histogram, our dataset is  imbalanced , likely having a bias towards English. We decided to sample 50,000 sentences from each language to even it out.

Some languages, however, had a total of fewer than 50,000 sentences, like Romani, so we included all the sentences of those languages. This is shown in the second histogram.

This process trimmed the number of sentences down to 455,910 sentences.

Pipeline

This is the process that we used to create our language detection project.

obtaining a dataset

Our first step was to find a  dataset that contained sentences in various different languages. We wanted to find a dataset with a large number of sentences for each language so that it could effectively be used to train the model to differentiate between languages.

data cleaning and sampling sentences

From our dataset, we removed all the sentences that were not within a specified threshold based on character count. From eleven languages that we decided to include, we selected a large number of sentences from each. Then, we split our data into training, testing, and validation sets in a 7:2:1 ratio.

feature engineering

We built a list of trigrams from our data so that the model would be able to identify different features in the sentences. We collected the 200 most frequent trigrams from each language and removed any overlaps to create a set of more than 1300 trigrams in total.

vectorization

After creating our set of trigrams, we needed to count the number of occurrences of each trigram in each sentence to convert the sentences to vectors. We used CountVectorizer from the Scikit-learn library to achieve this. Then, we normalized the data so that all the values were between zero and one.

training our model

We created a model and fed it our training data to teach it to distinguish between different languages. Using one-hot encoding, we created the expected outputs to be compared with the model's predictions. After training, we evaluated the model using the test data.

releasing our product

Our final step was to release our language identifier to the world so that users can input sentences that the computer then provides predictions for. To deploy our project to the web, we used many tools such as HTML, CSS, JavaScript, and Flask.

Model Structure

  • Our model is based on a Multi-layer Dense Neural Network .
  • It has three hidden layers as shown in the figure, with 500 neurons in each.
  • The input is a vector of 1391 dimensions.
  • The output is a vector of 11 dimensions/languages.
  • We used the module Keras from Python and built the model using Sequential and Dense Layers.
  • The optimizer used is Adam along with a cross-entropy loss.
  • With our loss rate changing and continuous training, we achieved a test accuracy of about 98%â€‹â€‹â€‹.

Evaluation

Once we were done training the model, we used our test data to evaluate its accuracy. 

T he accuracy turned out to be above 98% , meaning that our model can predict which language a sentence is with close to 100% accuracy. 
We also used a confusion matrix to help us understand and compare the accuracy of our model for different languages.
